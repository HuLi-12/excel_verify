# 基本信息表校验工具

本项目是本地命令行版 Excel 校验 MVP：

- 解析模板表和汇总表的多行表头、合并单元格
- 对比模板字段和汇总表字段
- 检查缺字段、多字段、字段顺序、空表头列有数据
- 按 YAML 规则检查必填、数值、枚举、手机号
- 输出保留原汇总表格式的 `校验错误报告.xlsx`
- 对错误单元格标红，并添加批注说明错误类型
- 支持自动修正、黄色警告、红色错误三类处理
- 附加 `错误明细`、`警告明细`、`自动修正明细` sheet

## 当前处理流程

工具不是直接拿原始单元格值做判断，而是按下面顺序处理：

```text
读取模板和待校验 Excel
  ↓
解析模板多行表头，生成字段路径
  ↓
根据模板示例值 + 字段名推断字段规则
  ↓
从待校验表中定位“序号=1”的真实数据起始行
  ↓
逐格读取原始值
  ↓
基础清洗
  ↓
格式规范化
  ↓
规则校验
  ↓
写出带标注的新 Excel
```

基础清洗会在校验前执行：

```text
1. 删除所有空白字符：半角空格、全角空格、Tab、换行、回车
2. 删除末尾中文句号 `。` 和英文句号 `.`
3. 将英文分号 `;` 统一为中文分号 `；`
```

例如：

```text
划 转。                 -> 划转
500 平方米。             -> 500平方米
2026 - 05 - 06          -> 2026-05-06
138 0000 0000           -> 13800000000
500;1200;1350           -> 500；1200；1350
```

如果清洗或规范化后可以通过校验，会记录为 `NORMALIZED`，写入 `自动修正明细`。

## 文件位置

默认输入路径：

```text
input/资产基本信息表(1).xlsx
input/汇总表截止20260416.xlsx
```

默认规则：

```text
config/asset_basic_info_rules.yaml
```

当前版本的规则主要是运行时从模板动态推断，不会每次强制保存一份新的规则文件。`config/asset_basic_info_rules.yaml` 用于维护人工配置、特殊空值、必填关键词、枚举补充等覆盖规则；更换模板后，程序会重新读取新模板并重新推断字段规则。

默认运行配置：

```text
config/app.yaml
```

示例：

```yaml
input_dir: input
template_path: input/基本信息表.xlsx
summary_path: input/汇总表.xlsx
rule_path: config/asset_basic_info_rules.yaml
output_dir: output
replace_error_values: true
generate_error_detail_sheet: true
```

默认输出会自动加时间戳：

```text
output/校验错误报告_YYYYMMDD_HHMMSS.xlsx
```

输出文件会复制待校验汇总表的原始工作表和格式，然后按等级处理：

```text
自动修正：写入规范化后的值，浅蓝底纹，记录到 自动修正明细
警告：保留原值，黄色底纹，记录到 警告明细
错误：按 replace_error_values 决定是否替换为 /，红色底纹，记录到 错误明细
```

如果 `replace_error_values: true`，错误单元格会替换为 `/`，并设置水平居中、垂直居中；如果设为 `false`，错误单元格保留原始值只标红。
如果 `generate_error_detail_sheet: true`，会生成明细 sheet；如果设为 `false`，不生成错误/警告/自动修正明细。
因为默认输出带时间戳，通常不会被 Excel 或 WPS 的文件占用影响；如果通过 `--output` 指定固定文件名，请先关闭同名文件。

## 校验规则说明

字段规则来源按优先级理解为：

```text
模板示例值 + 字段名自动推断
  ↓
YAML 人工规则补充或覆盖
```

当前支持的主要规则：

```text
string：普通文本
positive_integer：正整数
number：非负数字，允许小数
number_with_unit：数字 + 指定单位
enum：枚举值
list：中文分号分隔的列表
date：日期格式识别和统一
phone：手机号
id_card：身份证号
any：不做强校验
```

面积类字段会按 `number` 处理，即使模板示例是整数，真实数据也允许填写正小数。例如 `有产权面积`、`建筑面积（平方米）` 这类字段不会被限制成正整数。

### 列表字段

模板示例中包含中文分号 `；` 时，会推断为列表字段。列表子类型会继续根据每一项推断，例如：

```text
500；1200；1350               -> 数字列表
500平方米；1200平方米          -> 带单位数字列表
3号线上沙沟站；2号线青山路口站  -> 字符串列表
```

列表校验规则：

```text
英文分号 ; 会自动转为中文分号 ；
中文逗号、英文逗号、顿号会标为黄色警告
列表中混入“无”，例如 500；无；1350，会判为红色错误
整个字段填“无”，会标为黄色警告，保留原值
```

### 日期字段

日期字段会根据模板示例推断目标格式：

```text
2026.05.06 -> yyyy.MM.dd
2026-05-06 -> yyyy-MM-dd
20260506   -> yyyyMMdd
```

待校验数据支持常见日期格式，只要能解析为真实日期，就会统一成模板格式；非法日期会判为错误。

### 身份证和手机号

字段名包含以下关键词时，会进入特殊校验：

```text
身份证字段：身份证、身份证号、证件号码、居民身份证
手机字段：手机、手机号、联系电话、联系人电话、电话
```

身份证号规则：

```text
18 位
前 17 位为数字
最后 1 位为数字或 X/x
小写 x 会自动规范化为大写 X
```

手机号规则：

```text
11 位数字
以 1 开头
第二位为 3-9
```

## 运行

```bash
python main.py
```

路径优先级：

```text
命令行参数 > config/app.yaml > 自动扫描 input/
```

临时换一个汇总表：

```bash
python main.py --summary "input/新的汇总表.xlsx"
```

显式指定全部路径：

```bash
python main.py --template "input/资产基本信息表(1).xlsx" --summary "input/汇总表.xlsx" --rules "config/asset_basic_info_rules.yaml" --output "output/校验错误报告.xlsx"
```

如果 `config/app.yaml` 不存在或没有配置模板/汇总表路径，程序会自动扫描 `input/`：

- 文件名包含 `基本信息表` 或 `模板` 的 Excel 作为模板
- 文件名包含 `汇总表` 或 `汇总` 的 Excel 作为待校验数据
- 自动忽略 Excel/WPS 生成的 `~$` 锁文件
